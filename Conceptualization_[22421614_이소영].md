# OSS-Design-Project

## 1. Business purpose
### 1) Project background
현대인들은 신용카드, 체크카드 더불어 요즘은 간편결제 서비스로 삼성페이, 카카오페이 등 휴대폰만 가지고있다면 어디를 가던 쉽게 결제를 할수있어서 편리하게 사용하며 소비활동을 하고있습니다. 하지만 결제방식이 간단해짐에 따라 스스로 얼마를 어디에 썼는지 체계적으로 파악하기 어려워졌으며, 과도한 지출이 습관화된 경우, 소득이 있더라도 지출이 자금이 어디로 유출되는지 파악하지못해 경제적 어려움을 겪는 사례가 많습니다. 이러한 지출 습관을 교정하고 사용자가 돈을 체계적으로 관리할 수 있도록 돕기위해 프로그램을 구상하였습니다.

이 프로그램은 단순히 소득과 지출을 기록하는 것을 넘어, 은행앱과의 연동, 더치페이 계산, 분야별 지출 분석 기능을 제공하여 사용자의 소비 패턴을 시각화하여 사용자가 어떤 분야에서 지출이 많은지 파악하여 특정분야의 지출을 줄이는데 도움을 줍니다. 또, 사용자가 이번달 지출가능금액을 설정하여 금액을 초과하경우 경고알림을 주는 방식 등으로 자산관리에 도움을 줍니다. 또한, 월급이 정해진 날에 들어왔는지 확인가능하게 사용자가 지정한날짜에 매달 “~일 월급은 들어왔나요? 확인해보세요!” 등의 알림을 지원하여 사용자의 소득이 들어오지않았는지 한번 더 확인가능하도록 하는 기능을 지원하는 것을 목표로 한다. 
 따라서 과도한 지출이 습관화 된 사람들과 지출관리를 효율적으로 하고자하는 사람들이 이 앱의 타겟층이 될 수 있다.
### 2) Goal
가계부관리에 관한 종합 서비스 어플리케이션 개발
은행데이터 연동을 통한 지출습관의 분석 및 교정
사용자별 맞춤형 예산 설정 및 과소비 방지 알림 시스템 개발
사용하기쉽게 GUI활용
### 3) Target Market
과도한 지출로 인한 체계적인 자금관리가 필요한 사람
스스로 소비패턴으 파악하지 못해 돈관리에 어려움을 겪는 사람

## 4. Concept of operation

### 1) 금융 데이터 연동
| Purpose | 사용자가 수기로 입력하는 번거로움을 제거하고, 모든 소비 내역을 누락 없이 관리하기 위함 |
| :--- | :--- |
| Approach | 사용자의 동의하에 은행 API를 연동합니다 실시간으로 수집된 결제 데이터는 자동으로 가계부에 업로드됩니다 이때, 사용자가 원치 않는 내역이나 개인적인 거래는 직접 삭제하거나 메모를 추가할 수 있는 편집 권한을 부여합니다 |
| Dynamics | 새로운 결제가 발생하거나 사용자가 연동된 내역을 수정하고자 할 경우 |
| Goals | 은행 내역을 가게부에 추가합니다 |

### 2) 상위 테마 그룹화
| Purpose | 파편화된 개별 지출을 하나의 큰 테마으로 통합하여 총지출액을 파악하기 위함 |
| :--- | :--- |
| Approach | 사용자가 특정 지출(티켓, 야구장 음식, 응원용품 등)을 선택한 후 특정 테템(야구장)이라는 상위 태그를 지정합니다 시스템은 동일한 태그가 부여된 모든 지출을 합산하여 별도의 테마별 소비 리포트를 생성합니다 |
| Dynamics | 사용자가 특정 장소나 이벤트에서 사용한 총 비용을 확인하고싶을 경우 |
| Goals | 사용자의 취미 및 활동 패턴에 따른 정교한 자산 분석이 가능합니다 |

### 3) 더치페이 지출 계산
| Purpose | 여럿이 함께 결제한 경우 본인의 실제 지출액만을 정확히 기록하기 위함 |
| :--- | :--- |
| Approach | 전체 결제 금액이 연동되면 사용자가 더치페이 기능을 실행하여 총 인원수를 입력합니다 예를 들어 30,000원을 3명이 나누었을 경우, 시스템은 사용자의 실제 지출인 10,000원만을 지출 데이터로 반영합니다 |
| Dynamics | 지인과의 공동 소비 후 개인 지출분을 정산이 필요할 경우 |
| Goals | 정산 과정에서 발생하는 가계부 데이터의 오차를 차단합니다 |

### 4) 가용 금액 경고 알림
| Purpose | 사용자가 설정한 예산 내에서 지출하도록 돕기 위함 |
| :--- | :--- |
| Approach | 사용자가 월 수입과 고정 지출을 입력하면 시스템이 한 달간 써도 되는 금액을 설정합니다 일일 혹은 월별 지출이 예산의 80%를 초과할 경우 시스템은 경고 알림을 발송합니다 |
| Dynamics | 가용금액의 80%넘어갈 경우  |
| Goals | 사용자의 지출 습관을 능동적으로 교정하고 계획적인 소비를 유도합니다 |

### 5) 고정 소득 및 지출 확인 알림
| Purpose | 월급이나 수입이 제때 들어왔는지 확인하여 소득 관리의 누락을 방지를 위함 |
| :--- | :--- |
| Approach | 사용자가 매달 월급날을 지정하면, 해당 날짜에 시스템이 "n일 월급은 들어왔나요? 확인해보세요!"라는 푸시 알림을 보냅니다 사용자가 확인 버튼을 누르면 연동된 계좌의 입금 내역을 즉시 조회하여 보여줍니다 |
| Dynamics | 사용자가 설정한 특정 날짜(월급일, 적금일 등)가 되었을 경우 |
| Goals | 사용자가 소득 유입 상태를 명확히 인지하고 다음 달 예산을 계획하는 기초 자료로 활용하게 합니다 |

### 6) 소비 습관 분석
| Purpose | 매일 가계부를 확인하는 습관을 형성하고 장기적인 소비 패턴을 개선하기 위함 |
| :--- | :--- |
| Approach | 지출이 잦은 저녁 시간대에 "오늘의 지출을 확인하고 테마별로 분류해 보세요"라는 알림을 제공합니다 또한 매월 초에는 지난달 대비 지출 증감 수치를 분야별 차트로 시각화하여 제공합니다 |
| Dynamics | 매일 설정된 시간과 한 달 주기가 완료되었을 경우 |
| Goals | 지속적인 가계부 작성을 유도하여 사용자가 자신의 소비를 스스로 통제할 수 있도록합니다 |

## 5. Problem statement

### 1) 데이터 보안 및 개인정보 보호
은행 API 연동은 사용자의 모든 금융 내역을 취급하므로, 해킹이나 데이터 유출 시 개인정보 침해 사고로 이어질 수 있습니다. 따라서 데이터 암호화 기술을 통해 보안성을 극대화해야 합니다.
### 2) 사용자의 지속성 및 편의성 확보
가계부 작성은 번거로운 작업으로 인식되어 사용자가 중도에 포기할 가능성이 높아 최소한의 터치로 모든 기능을 수행할 수 있는 직관적인 GUI 디자인이 필요합니다.

## 6. Glossary
| 용어 | 설명 |
| :--- | :--- |
| 상위 테마 | 개별 지출 내역들을 하나의 큰 주제(예: 야구장, 여행)로 묶어 관리하는 상위 카테고리 단위입니다. |
| 더치페이 | 전체 결제 금액을 함께한 인원수만큼 나누어, 사용자의 실제 부담액만을 산출하여 기록하는 기능입니다. |
| 가용 금액 | 사용자의 수입과 고정 지출을 계산하여, 현재 시점에서 추가로 지출 가능한 안전 예산 범위를 의미합니다. |
| 은행 API | 각 은행 및 카드사의 금융 데이터를 프로그램과 실시간으로 연동하기 위한 인터페이스 규격입니다. |
| 고정 지출 | 월급, 적금, 통신비 등 매달 정해진 날짜에 반복적으로 발생하는 수입 및 지출 항목입니다. |
